# WEEK 6: VPC WITH BASTION SERVER AND NAT GATEWAY

## Objective:

To design a secure custom VPC architecture with public and private subnets, secure SSH administration through a Bastion server, and outbound internet access for private servers via NAT Gateway.

## Theory Steps:

1. Login to AWS Console.
2. Create a custom VPC with CIDR block 10.0.0.0/16.
3. Create:

   * Public Subnet (10.0.1.0/24)
   * Private Subnet (10.0.2.0/24)
4. Create and attach an Internet Gateway to the VPC.
5. Configure a Public Route Table:

   * Route 0.0.0.0/0 → Internet Gateway
6. Associate Public Subnet with Public Route Table.
7. Create security groups:

   * BastionSG: SSH from admin IP
   * DatabaseSG: SSH/MySQL only from Bastion subnet
8. Launch Bastion Server in Public Subnet.
9. Launch Database Server in Private Subnet without public IP.
10. Transfer private key from local machine to Bastion server.
11. SSH from local machine → Bastion → Private DB server.
12. Verify private server cannot access internet initially.
13. Create NAT Gateway in Public Subnet.
14. Allocate Elastic IP to NAT Gateway.
15. Create Private Route Table:

* Route 0.0.0.0/0 → NAT Gateway

16. Associate Private Subnet with Private Route Table.
17. Verify private server internet access using apt update.
18. Cleanup all resources.

## Key Concepts:

* VPC Isolation
* Bastion Architecture
* NAT Gateway
* Secure Administration
* Route Tables
